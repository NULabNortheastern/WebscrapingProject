Dublin Core is a set of metadata standards. It was created to reject siloed cultural memory and enable radically open cultural heritage data. Dublin Core is a set of fifteen "core" elements for describing resources. It is designed to be simple and flexible, which has both advantages and disadvantages. You can learn more about working with Dublin Core HERE.

Fun fact: it was named after Dublin, Ohio, not Dublin, Ireland!

Best Practices

  • All the fields are optional, so you can choose which you want to use to describe the items in your archive; be strategic about which fields you select!
  • It is essential to have consistency in metadata fields across your items: both in which you select and in how you fill those in. For instance, you don't want to have "Firstname Lastname" in half your "creator" fields and "Lastname, Firstname" in the other half.
