Adding Media and Images to KnightLab StoryMap

  • Use the “Media” section to add media to a slide. You can upload an image or put a URL of an image OR a video (like a YouTube video). If you want to add an image from a URL, just make sure the link you use is for the image file itself (not just a webpage with the image on it)
  • If you have an image or a video in the “Media” section, but want to include an image in the text box, you can! The text box reads basic HTML. 

Using the <img> tag in StoryMap

  • Use the HTML <img> tag to insert the image following this format: <img src="URL"> 
  • The <img> creates a holding space for the referenced image 
  • src specifies the path to the image 
  • Fill in the URL of the image within the quotation marks (replacing the text that says URL but without deleting the quotation marks)

Embed Media and Images on a Slide

  1. Click the </> button (turn on HTML)  
  2. Get a URL of your image. Wikimedia Commons is a great source for images. Make sure to use the embed file code--don’t forget to cite!
  3. Use the HTML <img src="URL"> template to insert the image.
  4. Check the “Preview” to see the image

Using HTML to Customize StoryMap

Change the text color in StoryMap

To change the text color in KnightLab StoryMaps you’ll need to use the <span> tag in HTML.

Click the HTML button (</>) in the editor. Then add the tag; for example: <span style ="color:red"> YOUR TEXT HERE </span>

Note: you must open and close the tags appropriately for the code to work. 

Be sure to wrap the <span> tag around the entire text you wish to be in a different color. 

Experiment with different colors! Try it on your own by changing the color from “red” to “purple”. 

Add alternative text for accessibility purposes

You can add alt text with HTML code, using the <img src =  "IMAGE LINK" alt = "IMAGE DESCRIPTION" > tag to make the image accessible for screenreaders.  

Note: typically, the <img> tag is associated with a specific image. However, the software for StoryMap separates the slide image from the caption. Be aware of this as you make accessibility-related improvements to your StoryMap.  

Alternatively, you can do longer-form, more detailed image descriptions within the caption section of the image slide. This may be the best approach considering software limitations.
