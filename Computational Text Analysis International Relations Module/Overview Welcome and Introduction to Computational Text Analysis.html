Overview

Welcome, to the Digital Integration Teaching Initiative's (DITI) module on web-based computational text analysis. In this module, we will introduce you to the following:

  • Important vocabulary words and definitions in computational text analysis
  • Best practices for building a corpus and preparing the text for analysis
  • User-friendly web-based text analysis tools

 

Introduction to Computational Text Analysis

Computational text analysis refers to an array of methods that can be used to “read” texts with a computer. This form of analysis can range from basic word frequency counts to more advanced techniques like machine learning. Text analysis is often used on a corpus, or a collection of multiple texts, and provides a glimpse into patterns across the texts. Some people also perform text analysis on larger individual documents, like novels or autobiographies.

Why?

Computational text analysis can help us analyze very large amounts of data and discover patterns in texts. Particular disciplines care deeply about the language used and how this language may reach intended audiences. Text analysis provides another method for approaching these discourses.

How?

Computational text analysis can be conducted using web-based tools or coding languages like Python and R. Many web-based tools are open-access and usually free as long as you have access to a computer and the internet. These tools have limits on how much data they can handle but are usually useful for small projects. Some of these include Voyant, Data Basic.io tools, and Jason Davies’ Word Tree tool.

 

Learning Objectives

By the end of this module, you should be able to:

  • Understand  basic vocabulary around doing text analysis, such as "corpus" and "stop words"
  • Build a small textual corpus and prepare the text for analysis
  • Use a variety of web-based tools for text analysis and interpret the results
